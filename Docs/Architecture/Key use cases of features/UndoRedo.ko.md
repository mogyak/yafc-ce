이 문서는 [영어](UndoRedo.md)로도 볼 수 있습니다.

Undo와 Redo 기능은 다음 핵심 사용 사례에서 일반적인 작업을 크게 개선합니다.

핵심 사용 사례:
*소리 없이 깨진 시트를 고치는 데 도움*. 명시적인 오류는 없지만 링크 설정이나 과잉 생산 미지정 때문에 결과가 비정상적이면 시트가 조용히 깨진 상태가 됩니다. 이 문제는 훨씬 나중에 알아차리면 무엇이 깨졌는지 명확하지 않아 더 심각해질 수 있습니다. Undo가 없다면 큰 시트를 고치기 위해 전체 구간을 처음부터 다시 만들어야 할 수 있습니다. 시트가 클수록 solver가 어디에서 문제를 겪는지 명확히 알려주지 않기 때문에 디버깅이 더 어렵습니다. 이런 상황에서 Undo는 정확히 어느 변경이 시트를 깨뜨렸는지 알려주어 많은 시간을 아낄 수 있습니다.

덜 흔하지만 여전히 중요한 사용 사례:
*같은 산출물에 대한 두 레시피 비교*. 생산 체인에서 같은 산출물의 두 레시피를 비교하려면 둘 다 시트에 추가한 뒤, 둘 다 활성화되어 있을 때 사용되는 레시피의 "레시피 활성화" 설정에 대해 Undo/Redo를 반복하면 됩니다. 이 사용 사례의 대안은 시트를 복제하고 두 설정을 별도 시트에서 구성하는 것이지만, 시간이 더 걸리고 차이를 발견하기도 더 어렵습니다.
